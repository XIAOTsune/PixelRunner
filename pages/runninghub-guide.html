RunningHub 插件 5 分钟上手

这份教程适合第一次使用插件时快速过一遍核心流程。你只需要完成 API Key 保存、应用解析、图片获取和任务运行这四步，就可以把整条链路跑通。

本地 HTML 教程 适合新用户 覆盖常见问题
1. 准备 API Key 先在 RunningHub 账号里拿到可用的 API Key，并保存到插件设置里。
2. 解析应用 把应用 ID 或链接填进设置页，保存成你常用的 AI 应用卡片。
3. 进入 Workspace 选择应用、准备图片和提示词，然后直接提交运行。

标准流程

01

保存 API Key

在“系统设置”里填入 RunningHub API Key，保存后插件会记住这个值，后续运行任务时会自动读取。

02

解析并保存应用

把 RunningHub 应用链接或应用 ID 粘贴到设置页，解析成功后保存。这样 Workspace 里就能直接选择，不用每次重新输入。

03

准备输入内容

回到 Workspace，选择一个已保存应用。根据应用需要上传图片、补充提示词、调整参数。

04

运行并查看日志

点击运行后，重点看状态摘要和日志区域。任务完成后结果会自动下载并回贴到 Photoshop。

新手最常遇到的 4 个点

  • 应用链接和应用 ID 都可以 设置页支持直接粘贴完整 RunningHub 链接，插件会自动规范化成可解析的应用 ID。
  • 提示词不宜过长 单条提示词建议控制在 4000 字符内，过长可能被 RunningHub 侧拒绝或让调试变难。
  • 并发数不是越高越好 默认并发数已经设置为 3。只有在账号本身支持更高并发时，再考虑继续调大。
  • 优先从已保存应用开始 这样能减少解析链路的不确定性，也更适合稳定复用你常跑的工作流。

排查建议

任务没有开始运行： 先检查 API Key 是否已保存、应用是否解析成功、图片输入是否完整。
返回结果不符合预期： 优先确认当前应用、提示词和图片是否都是最新的，再观察日志里是否出现旧输入残留。
看起来有点卡： 优先留意日志刷新、图片体积和并发数设置。并发数调得过高不一定更快，反而可能让体验变差。
