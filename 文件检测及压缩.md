# 文件检测及压缩

## 1. 分阶段执行方案

### 阶段 A：可视化与预检（先挡风险）
- 捕获后写入 `sourceMeta/uploadMeta`，前台仅展示基础信息（mime、bytes、分辨率、bitDepth），不展示 safe/risky/blocked 风险标签。
- 运行前执行 preflight gate：
  - `safe` 直传。
  - `risky/blocked` 进入自动压缩链路后再上传。
  - 自动压缩失败或压缩后仍 `blocked` 时本地阻断。
- 位深只提示，不作为唯一阻断条件。

### 阶段 B：自动压缩接入（原生导出链路）
- 仅使用 Photoshop 原生导出链路（`batchPlay save`）做压缩，禁止 JS decode/canvas 压缩。
- 压缩策略采用“质量梯度 + 长边梯度”：
  - 质量梯度：`[10, 8, 7, 6, 5, 4]`
  - 长边梯度：`[原始长边, 6144, 5120, 4096, 3072, 2560, 2048]`
- 增加保护：`maxAttempts=18`，`maxCompressionDurationMs=12000`。

### 阶段 C：上传硬校验与可观测性
- 上传前再次执行 bytes 硬校验（`bytes > hardLimitBytes` 本地失败）。
- 输出结构化上传诊断日志：
  - `sourceBytes/uploadBytes`
  - `sourceMime/uploadMime`
  - `sourceSize/uploadSize`
  - `sourceBitDepth/uploadBitDepth`
  - `riskBefore/riskAfter`
  - `compressionApplied/quality/maxEdge/attempts/durationMs`
  - `endpoint/httpStatus/apiCode/apiMessage`
