# UI 与性能优化执行方案

## 目标

- 提升插件界面的整体质感，让 Workspace、Tools、Settings 三个页面更统一、更像一套完整产品。
- 消除当前“轻微卡顿”和“切换偏硬”的体感问题，优先优化日志区、任务状态区、列表弹窗与关键交互过渡。
- 在不改业务流程和 DOM 契约的前提下，优先做低风险、高体感收益的优化。

## 当前问题

### UI 问题

- 页面已有基础视觉统一，但层级感仍偏硬，切换时缺少过渡，界面反馈显得“直接跳变”。
- Tab、Modal、图片捕获等关键操作缺少轻量动画，用户很难感知“状态正在切换”。
- 交互反馈已有颜色变化，但缺少进入、激活、确认成功等过渡细节。

### 性能问题

- 任务状态摘要刷新频率过高，容易在宿主环境里造成持续重绘。
- 日志区域采用整段文本重写和重复滚动校正，日志增多后更容易出现顿感。
- 应用选择器 / 模板选择器每次操作都做全量 `innerHTML` 重绘，存在不必要的 DOM churn。
- 部分界面交互仍使用硬切换方式，视觉上会放大“卡”的主观感受。

## 设计方向

- 视觉方向采用“开发者工具 / 生产力面板”风格。
- 保持深色工作台基调，统一强调色为蓝色系，减少无效装饰。
- 动效策略采用“少而准”的微交互：
  - Tab 切换：轻量淡入上移
  - Modal 打开：遮罩淡入 + 面板轻微上浮
  - 按钮 / 列表项：按压反馈、激活态更明确
  - 图片捕获：成功后的状态反馈更自然
- 避免大范围复杂动画，只动画 `opacity`、`transform`、`border-color`、`background-color`。

## 执行分期

### 第一阶段：高收益性能修正

- 降低任务摘要 ticker 刷新频率。
- 为任务摘要增加“内容未变化不重绘”保护。
- 优化日志追加策略，避免每次都整段读取再回写。
- 合并日志滚动跟随逻辑，减少重复滚动校正。
- 为应用 / 模板弹窗列表增加“相同内容不重复写入”的保护。

### 第二阶段：关键过渡补齐

- 为 Tab 内容切换增加进入动画。
- 为通用 Modal 和打赏弹窗增加打开动画。
- 收敛 `transition: all`，改成只针对必要属性过渡。
- 强化卡片、可点击项、图片捕获容器的悬停与按压反馈。

### 第三阶段：界面精修

- 统一主次按钮、标题层级、卡片边框与间距。
- 优化 Workspace 的阶段感与状态反馈信息层级。
- 继续检查局部组件是否存在视觉密度不一致的问题。

## 本轮落地范围

- 新增本执行方案文档。
- 任务摘要刷新降频与去重渲染。
- 日志区域追加与滚动策略优化。
- 应用选择器 / 模板选择器局部渲染优化。
- Tab / Modal / 部分卡片的轻量过渡动画补齐。

## 验收标准

- 切换 Tab、打开弹窗、滚动查看日志时，主观体感更顺，不再有明显“硬切”。
- 运行任务时任务摘要更新稳定，不出现高频闪动。
- 日志持续输出时，界面响应明显更平滑。
- 现有控制器绑定 ID、核心 DOM 契约、业务行为保持不变。
