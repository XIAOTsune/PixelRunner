# PixelRunner 快捷入口功能设计讨论

更新时间：2026-04-19

## 已确认方向

新增“快捷入口”能力：用户先在 Photoshop 中框选区域，然后在 PixelRunner 里点击收藏好的快捷入口，插件自动使用该入口保存的 RunningHub 应用和输入参数运行任务。

这个功能的目标不是增加一个复杂管理系统，而是把用户高频、固定参数的工作流折叠成一个按钮。

## 已确认决策

1. 记住快捷入口模式。  
   用户切到快捷入口模式后，下次打开插件或刷新工作台时仍保持快捷入口模式。

2. 无选区时不默认使用整图。  
   如果快捷入口需要捕获 Photoshop 当前选区，但没有检测到选区，则弹出提醒，不直接用整图。用户如果想跑整图，可以自己框一个覆盖整图的选区。

3. 预置快捷入口改为 JSON 包交换方案。  
   不在代码里固定一批预置快捷入口。后续通过统一 JSON 导入/导出，让用户可以交换：
   - 快捷入口设置
   - RunningHub 应用卡片
   - 提示词卡片

   这样后续维护预置内容时，只需要提供一份 JSON 文件。

4. 快捷入口保存全部非图片参数。  
   第一版不做参数勾选、不做部分参数锁定。保存时直接记录当前应用中所有非图片输入项的值。

5. 快捷入口不允许运行前临时改 prompt。  
   点击即跑。需要不同 prompt 或不同参数时，用户新增一个不同名字的快捷入口；旧入口可以删除。

6. 从当前应用添加到快捷入口需要一个短按钮。  
   按钮名称定为 `快捷`，悬停提示为“把当前应用和参数保存为快捷入口”。点击按钮后弹出小弹窗，只填写快捷入口名字即可保存。

## 核心概念

快捷入口不是新的 RunningHub 应用类型，也不是提示词模板。

它是一个保存快照：

```text
快捷入口 = 应用引用 + 非图片参数 + 图片捕获策略 + 显示名称
```

快捷入口不保存：

- API Key
- 图片本体
- base64
- 任务结果
- Photoshop 文档内容

## 用户流程

### 创建快捷入口

1. 用户选择一个普通 RunningHub 应用。
2. 用户填写好参数，例如 prompt、强度、尺寸、模式等。
3. 用户点击“添加到快捷入口”。
4. 弹出命名弹窗。
5. 用户输入名称，例如“一键磨皮”。
6. 保存。

弹窗建议：

```text
添加到快捷入口

名称：[一键磨皮                ]

将保存当前应用和全部非图片参数。
运行时会要求先框选 Photoshop 区域，再自动捕获选区提交。

[取消] [保存]
```

第一版只填写名称，不在弹窗里展示完整参数，避免打断。

### 使用快捷入口

1. 用户在 Photoshop 中框选区域。
2. 打开 PixelRunner。
3. 进入快捷入口模式。
4. 点击某个快捷入口按钮。
5. 插件检查选区。
6. 捕获选区并写入快捷入口绑定的图片字段。
7. 合并快捷入口保存的参数。
8. 提交 RunningHub。
9. 进入现有任务卡追踪和自动贴回流程。

如果没有选区：

```text
未检测到 Photoshop 选区。
请先框选要处理的区域，再运行“一键磨皮”。

[知道了]
```

不提供“改用整图继续运行”，避免误操作。用户确实要整图时，可以框选整图。

## 入口位置

### 应用选择卡片顶部

保留你的设想：在应用选择卡片顶部，把快捷入口作为一个特殊入口展示。

```text
选择应用

[ 快捷入口 ]
[ 应用 A ]
[ 应用 B ]
[ 应用 C ]
```

点击 `快捷入口` 后，工作台切换到快捷入口模式。

### 快捷入口模式显示内容

快捷入口模式不显示完整动态表单，只显示快捷入口列表和必要操作。

```text
快捷入口

[ 一键磨皮        运行 ]
[ 局部重绘        运行 ]
[ 高清放大        运行 ]

[ 管理导入导出 ]
```

每个入口可以是一张小卡片，也可以是紧凑按钮。第一版建议小卡片：

```text
一键磨皮
人像磨皮 Pro · 需要选区
[运行] [删除]
```

### “添加到快捷入口”按钮放在哪里

这个按钮要在普通应用模式下出现，而不是快捷入口模式里。

推荐位置：应用卡片工具栏的次级按钮区。

当前应用卡片大概是：

```text
选择应用
[切换] [刷新]
当前应用信息
```

建议变成：

```text
选择应用
[切换] [刷新] [快捷]
当前应用信息
```

理由：

- 用户选择好应用并填参数后，这个按钮语义自然。
- 不占动态表单空间。
- 和应用相关，不会误解为模板功能。
- 入口足够近，但不是主按钮，不会干扰运行。

按钮悬停提示：

```text
把当前应用和参数保存为快捷入口
```

备选位置：动态表单标题区右侧。

```text
输入参数                         [快捷]
```

这个位置也合理，因为保存的是当前参数。但它可能让表单区变得更重。优先建议放应用卡片工具栏。

## 模式状态

需要引入工作台模式：

```js
workspaceMode: "app" | "quick"
```

当用户选择普通应用时：

```js
workspaceMode = "app"
currentAppId = "..."
```

当用户选择快捷入口时：

```js
workspaceMode = "quick"
```

因为已经确认“记住快捷入口模式”，需要把这个模式持久化：

```text
pixelrunner.workspaceMode
```

或放进现有设置快照里。

建议同时保留最近真实应用：

```text
pixelrunner.currentAppId
pixelrunner.workspaceMode
```

这样用户从快捷入口模式切回普通应用时，还能回到上次真实应用。

## 快捷入口数据结构

建议存储：

```js
{
  version: 1,
  entries: [
    {
      id: "quick-...",
      title: "一键磨皮",
      appRef: {
        savedAppId: "local-app-id",
        appId: "runninghub-webapp-id",
        appName: "人像磨皮 Pro"
      },
      inputValues: {
        prompt: "自然磨皮，保留五官细节和皮肤纹理",
        strength: 0.65,
        size: "1024"
      },
      imageBindings: [
        {
          inputKey: "image",
          source: "selectionRequired"
        }
      ],
      meta: {
        createdAt: 0,
        updatedAt: 0,
        lastRunAt: 0,
        runCount: 0
      }
    }
  ]
}
```

### imageBindings

第一版建议默认：

```js
source: "selectionRequired"
```

含义：

- 运行时必须检测到 Photoshop 选区。
- 没有选区就提示用户先框选。
- 不自动改用整图。

如果一个 RunningHub 应用有多个图片字段，第一版可以先选择第一个必填图片字段；如果没有必填图片字段，再选择第一个图片字段。

多图片字段的精细绑定可以放到第二版。

## 快捷入口存储

新增独立存储 key：

```text
pixelrunner.quickEntries.v1
```

不要混入：

- `apps`
- `templates`
- `settings`

原因：

- 快捷入口不是应用，也不是提示词模板。
- 独立存储更方便清理、迁移、导入导出。
- 出错时可以只重置快捷入口，不影响用户已保存应用和模板。

## 统一 JSON 导入导出

这是本次新确认的重点。

当前用户已经可以通过 JSON 导入导出提示词卡片。建议升级为“PixelRunner 资料包”导入导出。

一个 JSON 包可以同时包含：

- 应用卡片
- 提示词卡片
- 快捷入口设置

### JSON 包建议结构

```js
{
  "schema": "pixelrunner.bundle",
  "version": 1,
  "exportedAt": "2026-04-19T00:00:00.000Z",
  "name": "小T常用修图快捷包",
  "apps": [],
  "templates": [],
  "quickEntries": []
}
```

### 导出逻辑

设置页可以把原来的模板导出升级为：

```text
导出资料包
```

可选第一版简单点：

- 导出全部应用卡片
- 导出全部提示词卡片
- 导出全部快捷入口

第二版再考虑勾选导出范围。

### 导入逻辑

导入时识别：

- 旧格式：只有 templates，则按旧提示词模板导入。
- 新格式：`schema === "pixelrunner.bundle"`，则导入 apps、templates、quickEntries。

导入策略定为同名覆盖、无同名追加：

- 不删除现有应用卡片。
- 不删除现有提示词卡片。
- 不删除现有快捷入口。
- 应用卡片按应用名称覆盖同名项。
- 提示词卡片按模板标题覆盖同名项。
- 快捷入口按快捷入口名称覆盖同名项。
- 无同名时追加为新项目。
- 遇到 ID 冲突但不同名时生成新 ID。
- 第一版不做同步、不做“清空后导入”。

第一版统一策略：

```text
导入时同名覆盖、无同名追加，不删除其他内容；不同名但 ID 冲突时生成新 ID。
```

导入完成提示：

```text
已导入：应用新增 3 个、覆盖 1 个；提示词新增 8 条、覆盖 2 条；快捷入口新增 5 个、覆盖 1 个。
```

### 预置快捷入口的新维护方式

不再把预置快捷入口写死在源码中。

后续你要发布预置内容时，可以提供一个 JSON：

```text
PixelRunner-常用快捷入口包.json
```

用户导入后获得：

- 你预置的应用卡片
- 对应提示词卡片
- 快捷入口按钮

这比“内置预置入口”更好维护，也更方便用户之间交换。

## 创建快捷入口时保存什么

点击“添加快捷入口”后保存：

- 当前应用的本地 id
- 当前应用的 RunningHub appId
- 当前应用显示名称
- 当前全部非图片参数
- 图片字段绑定策略：`selectionRequired`
- 快捷入口名称
- 创建时间和更新时间

不保存：

- 图片字段当前值
- 当前 Photoshop 截图
- API Key
- 任务结果
- 当前日志

## 编辑和删除

### 第一版编辑

第一版可以非常克制：

- 重命名
- 删除

不做：

- 参数编辑
- 图片字段绑定编辑
- prompt 临时修改
- 运行前参数弹窗

原因：你已经确认需要不同参数时新增一个不同名字的快捷入口即可。

### 后续编辑扩展

如果以后需要编辑参数，建议仍然不做单独参数编辑器，而是：

```text
用普通应用打开
```

打开后：

1. 切回普通应用模式。
2. 填入该快捷入口保存的参数。
3. 用户改参数。
4. 点击“更新快捷入口”或另存为新的快捷入口。

这个可以放到第二版。

## 快捷运行流程

```text
点击快捷入口
  -> 检查 API Key
  -> 检查快捷入口引用的 appId
  -> 获取或解析应用输入结构
  -> 检查 Photoshop 当前选区
  -> 无选区：提示并中止
  -> 有选区：捕获选区
  -> 合并应用默认值 + 快捷入口非图片参数 + 捕获图片
  -> 校验必填项
  -> 提交 RunningHub
  -> 进入现有任务卡
  -> 任务成功后沿用自动贴回逻辑
```

### 参数合并顺序

```text
应用默认值 < 快捷入口保存参数 < 运行时捕获图片
```

快捷入口保存参数覆盖应用默认值。运行时捕获图片覆盖所有旧图片值。

## 运行前交互

正常情况：不确认，点击即跑。

只有异常才打断：

- 没有选区。
- API Key 缺失。
- 快捷入口引用的应用不存在或 appId 无效。
- 应用输入结构变化，保存的 inputKey 找不到。
- 必填项缺失。

无选区时只提醒，不提供整图继续：

```text
未检测到 Photoshop 选区。
请先框选要处理的区域，再运行该快捷入口。
```

## 和现有工作台的关系

### 普通应用模式

保持现有动态表单。

新增按钮：

```text
快捷
```

推荐放在应用卡片工具栏：

```text
选择应用
[切换] [刷新] [快捷]
```

悬停提示：

```text
把当前应用和参数保存为快捷入口
```

点击后弹出命名弹窗，保存当前应用和全部非图片参数。

### 快捷入口模式

显示快捷入口列表，不显示完整参数表单。

建议结构：

```text
快捷入口
已收藏 5 个

[一键磨皮] [局部重绘] [高清放大]

导入/导出在设置页管理。
```

任务区仍复用现有任务卡。

## 第一版范围

第一版建议做：

1. 删除旧调研文档，保留当前设计文档。
2. 新增快捷入口独立存储 `pixelrunner.quickEntries.v1`。
3. 新增工作台模式 `app | quick`，并持久记住快捷入口模式。
4. 应用选择顶部增加 `快捷入口` 虚拟卡片。
5. 普通应用模式增加 `快捷` 按钮，悬停提示“把当前应用和参数保存为快捷入口”。
6. 点击后弹窗填写名称，保存当前全部非图片参数。
7. 快捷入口模式展示入口列表。
8. 点击入口后要求 Photoshop 必须有选区。
9. 捕获选区并直接运行。
10. 快捷入口支持删除和重命名。
11. 统一 JSON 资料包导入/导出，包含 apps、templates、quickEntries。

第一版不做：

- 运行前改 prompt。
- 快捷入口参数编辑器。
- 多步骤快捷链。
- 快捷入口分组。
- 快捷入口图标系统。
- 快捷入口云同步。
- 无选区时整图继续运行。
- 内置源码级预置快捷入口。

## 后续版本可以考虑

- 从快捷入口“用普通应用打开并编辑”。
- 更新现有快捷入口参数。
- JSON 导入时提供勾选范围。
- 快捷入口排序。
- 快捷入口分组。
- 快捷入口运行次数排序。
- 不同图片字段的手动绑定。

## 风险点和规避

### 1. 快捷入口变成第二套应用系统

规避：

- 第一版只允许命名、删除、重命名。
- 参数修改通过新增不同名字的快捷入口解决。
- 不做完整参数编辑器。

### 2. 用户误以为会自动整图运行

规避：

- 快捷入口卡片标注：`需要选区`。
- 没选区时中止并提示。
- 不提供整图继续按钮。

### 3. JSON 导入造成重复内容

规避：

- 第一版导入不覆盖，冲突生成新 ID。
- 导入结果显示数量。
- 后续再加选择性导入和覆盖策略。

### 4. 应用输入结构变了

规避：

- 保存快捷入口时记录 inputKey。
- 运行前检查 inputKey 是否存在。
- 失效时提示重新用当前应用参数创建快捷入口。

### 5. 多图应用无法准确绑定

规避：

- 第一版只自动绑定一个图片字段。
- 优先必填图片字段。
- 多图精细绑定放到后续版本。

## 当前推荐落点

最小可用版本应该像这样：

```text
普通应用模式：
  填好参数 -> 点“快捷” -> 输入名字 -> 保存

快捷入口模式：
  框选 Photoshop 区域 -> 点快捷入口 -> 自动提交

设置页：
  导出资料包 / 导入资料包
  资料包同时包含应用卡片、提示词卡片、快捷入口
```

这个方案和现有 PixelRunner 的主流程兼容，同时给高频用户一条非常短的路径。
