# PixelRunner 2.4.0 注意事项

## 1. 架构原则
- 当前版本已经切换到 `UXP Host Shell + WebView UI` 架构。
- 后续开发必须继续坚持前后端分离和模块化。
- 不要把复杂 UI 或业务逻辑重新塞回 Host。

### WebView 负责
- UI 展示
- 页面状态
- 表单交互
- 设置页交互
- 工作台交互
- 任务参数组织

### Host 负责
- 挂载 WebView
- bridge 通信
- 本地存储
- RunningHub 网络请求
- Photoshop / UXP 能力调用

## 2. 正式源码位置
- 正式 Host 源码目录：
  - `src/host`
- 正式 WebView 源码目录：
  - `src/webview`
- 构建入口：
  - `src/host-entry.js`
  - `src/webview-entry.js`

## 3. 必须遵守
- 不要直接修改 `dist/*.bundle.js`
- 不要把复杂逻辑重新塞回宿主层
- 不要回退到根目录旧源码做主开发
- 浏览器预览时直接打开 `app.html`
- 不要在浏览器里直接打开 `index.html`
- 功能对齐以 `PixelRunnerV2.3.3` 为准
- 不要再参考 `HHPS v1.3.1`

## 4. 根目录旧源码说明
- 根目录旧的 `app-*.js`、`index.js`、`host-photoshop.js` 不再是正式入口。
- 旧文件已归档到：
  - `legacy/root-src`
- 后续改动必须优先落在 `src/webview` 和 `src/host`

## 5. 当前工作台设计
- 已取消手动“放回 Photoshop”按钮。
- 当前设计为任务完成后自动贴回 Photoshop。
- 自动贴回逻辑：
  - 记录任务发起前的源文档信息
  - 默认允许跨文档回到源文档
  - 如果源文档存在原选区，则按原选区 bounds 自动贴回并加遮罩
  - 如果源文档没有选区，则自动回贴到源文档
- 这条链路已迁移并构建通过，但真实 Photoshop 行为仍需继续在插件环境中验证。

## 6. 当前设置页状态
- 设置页结构已继续向 `2.3.3` 收敛，但还不是最终版。
- 已完成或接入的内容：
  - API Key / 轮询 / 超时设置读写
  - 账户状态读取
  - 应用管理内联编辑区
  - 应用列表搜索 / 排序第一版
  - 应用远程解析链路第一版
  - 最近一次解析调试记录读取入口
  - 提示词模板轻量编辑区
  - 高级设置折叠栏
  - 环境诊断折叠栏
- 仍需继续向 `2.3.3` 收敛交互细节和文案。

## 7. 应用解析现状
- `2.4.0` 现在不再只是前端提取 `appId`。
- 已补齐 Host 侧远程解析链路：
  - WebView 调 `runninghub.parseApp`
  - Host 访问 RunningHub 解析接口
  - 自动归一化应用名、描述、输入结构
  - 解析结果回填到设置页编辑区
  - 保存后落库并同步到工作台
- 已新增解析调试存储：
  - `rh_last_parse_debug`
- 设置页环境诊断区可加载最近一次解析调试记录。
- 仍需继续完善：
  - 解析失败时更友好的界面反馈
  - 更贴近 `2.3.3` 的解析结果展示
  - 更稳的输入标签识别与字段提示

## 8. 提示词模板现状
- 已接入模板完整第一版链路：
  - 新建
  - 编辑
  - 删除
  - 列表
  - 导入 / 导出 JSON
  - 工作台模板选择器
  - 替换 / 追加写入
- 近期已补：
  - 模板模块主要文案乱码清理
  - 模板选择器搜索
  - 模板组合应用
  - 模板长度提示
- 后续仍需继续增强：
  - 更接近 `2.3.3` 的模板管理体验
  - 多模板组合策略细节
  - 模板应用后的更明确反馈

## 9. 图像输入链路现状
- 当前已完成第一版增强：
  - Photoshop 文档图像捕获
  - WebView 预览
  - 压缩参数控制
  - 多次捕获资产列表
  - 当前选中捕获图像
  - 字段级图像绑定
  - 删除捕获时清理字段绑定
  - 一键清空全部捕获
- 后续仍应继续提升：
  - 字段级上传策略可视化
  - 多图字段 / 数组字段支持
  - 更清晰的字段映射体验

## 10. 页面与样式注意事项
- 之前出现过 `app.html` 结构损坏、串位、坏闭合标签直接显示到界面的严重问题。
- 当前这类结构损坏问题已修复，但仍要警惕残留乱码和旧文案。
- 最近已修复一个布局问题：
  - 顶部三个页签下方内容会不同程度挤压标题栏
  - 已通过 `app.css` 调整导航、页头、内容区的垂直间距修复

### 遇到乱码时必须这样处理
- 优先检查：
  - `app.html`
  - `src/webview/*.js`
  - `app.css`
- 不要只看 `release` 或 `dist`
- 不要直接手改 `dist/app.bundle.js`
- 必须修源码后重新执行：
  - `npm.cmd run build`
  - `npm.cmd run package:test`

## 11. 构建与测试
- 可用命令：
  - `npm.cmd install`
  - `npm.cmd run build`
  - `npm.cmd run package:test`
- 当前测试包目录：
  - `release/PixelRunnerV2.4.0-test`
- 每次改动源码后都要同步执行：
  - `npm.cmd run build`
  - `npm.cmd run package:test`

## 12. 接手时优先顺序
1. 先阅读：
   - `docs/注意事项.md`
   - `docs/重构交接.md`
2. 继续以 `src/webview` 和 `src/host` 为正式源码开发
3. 不要回退到根目录旧源码做主开发
4. 继续把设置页、工作台交互向 `2.3.3` 收敛
5. 优先继续完善应用解析体验，不要退回只做前端壳子
6. 继续完善提示词模板体验和残留文案 / 乱码清理
7. 然后再继续提升图像输入链路与工程化

## 13. 一句话提醒
- 现在最重要的是“对齐 `2.3.3` 体验，同时守住 `2.4.0` 的前后端分离架构”，不要再把项目带回旧的混合式写法。
