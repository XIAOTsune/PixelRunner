# PixelRunner 2.4.0 注意事项

## 一、架构原则

- 当前版本已经重构为 `UXP Host Shell + WebView UI` 架构。
- 后续开发必须继续坚持前后端分离和模块化。
- 不要把复杂 UI 或业务逻辑重新塞回 Host。

### WebView 负责

- UI 展示
- 页面状态
- 表单交互
- 设置页交互
- 工作台交互
- 任务参数组织

### Host 负责

- 挂载 WebView
- bridge 通信
- 本地存储
- RunningHub 网络请求
- Photoshop / UXP 能力调用

## 二、正式源码位置

- 正式 Host 源码目录：
  - `src/host`
- 正式 WebView 源码目录：
  - `src/webview`
- 构建入口：
  - `src/host-entry.js`
  - `src/webview-entry.js`

## 三、必须遵守

- 不要直接修改 `dist/*.bundle.js`
- 不要把复杂逻辑重新塞回宿主层
- 不要回退到根目录旧源码做主开发
- 浏览器预览时直接打开 `app.html`
- 不要用浏览器直接预览 `index.html`

## 四、根目录旧源码说明

- 根目录旧 `app-*.js`、`index.js`、`host-photoshop.js` 不再是正式入口。
- 旧文件已归档到：
  - `legacy/root-src`
- 后续改动必须优先落在 `src/webview` 和 `src/host`。

## 五、功能对齐基线

- 当前功能对齐基线是：
  - `PixelRunnerV2.3.3`
- 不再参考：
  - `HHPS v1.3.1`

## 六、当前工作台结论

- 已取消手动“放回 Photoshop”按钮。
- 当前设计是：
  - 任务完成后自动贴回 Photoshop
  - 默认允许跨文档回到源文档
  - 如果源文档存在选区，则按原选区 bounds 自动贴回并加遮罩
  - 如果没有选区，则自动回贴到源文档

- 说明：
  - 这条链路代码已经迁入并完成构建验证
  - 但整体重构尚未完成，真实行为还需要继续在 Photoshop 中验证

## 七、当前设置页结论

- 当前设置页不是最终定稿。
- 用户已经明确指出：
  - `2.4.0` 当前设置页布局和交互，与原本 `2.3.3` 不一致。
- 这是成立的结论，后续需要继续回收敛。

## 八、当前提示词模板结论

- 当前 `2.4.0` 中：
  - 应用管理第一版已接通
  - 提示词模板功能没有完整迁完

- 提示词模板当前仍缺：
  - 新建
  - 编辑
  - 删除
  - 列表
  - 导入 / 导出完整逻辑
  - 工作台模板选择器

## 九、图像输入链路结论

- 当前已完成第一版增强：
  - Photoshop 文档图像捕获
  - WebView 预览
  - 压缩参数控制
  - 多次捕获资产列表
  - 当前选中图像
  - 分字段图像绑定
  - 删除捕获时清理对应字段绑定
  - 一键清空全部捕获

- 后续还应继续提升：
  - 字段级上传策略可视化
  - 多图字段 / 数组字段支持
  - 更清晰的字段映射体验

## 十、工具箱结论

- 当前已验证正常的工具：
  - 黑白观察层
  - 中性灰图层
  - 盖印图层
  - 高斯模糊
  - 锐化
  - 高反差保留
  - 内容识别填充
  - 选择并遮住

## 十一、构建与测试

- 当前可用命令：
  - `npm.cmd install`
  - `npm.cmd run build`
  - `npm.cmd run package:test`

- 当前测试包目录：
  - `release/PixelRunnerV2.4.0-test`

## 十二、接手时优先顺序

1. 先阅读：
   - `docs/注意事项.md`
   - `docs/重构交接.md`
2. 继续以 `src/webview` 和 `src/host` 为正式源码开发
3. 不要回退到根目录旧源码做主开发
4. 优先把设置页与工作台交互继续向 `2.3.3` 收敛
5. 优先补完提示词模板完整链路
6. 然后继续提升图像输入链路与工程化

## 十三、一句话提醒

- 现在最重要的是“对齐 `2.3.3` 体验，同时守住 `2.4.0` 的前后端分离架构”，不要再把项目带回旧的混合式写法。
