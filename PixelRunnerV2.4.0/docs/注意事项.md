# PixelRunner 2.4.0 注意事项

## 1. 架构原则
- 当前版本已经重构为 `UXP Host Shell + WebView UI` 架构。
- 后续开发必须继续坚持前后端分离和模块化。
- 不要把复杂 UI 或业务逻辑重新塞回 Host。

### WebView 负责
- UI 展示
- 页面状态
- 表单交互
- 设置页交互
- 工作台交互
- 任务参数组织

### Host 负责
- 挂载 WebView
- bridge 通信
- 本地存储
- RunningHub 网络请求
- Photoshop / UXP 能力调用

## 2. 正式源码位置
- 正式 Host 源码目录：
  - `src/host`
- 正式 WebView 源码目录：
  - `src/webview`
- 构建入口：
  - `src/host-entry.js`
  - `src/webview-entry.js`

## 3. 必须遵守
- 不要直接修改 `dist/*.bundle.js`
- 不要把复杂逻辑重新塞回宿主层
- 不要回退到根目录旧源码做主开发
- 浏览器预览时直接打开 `app.html`
- 不要在浏览器里直接预览 `index.html`
- 功能对齐以 `PixelRunnerV2.3.3` 为准
- 不再参考 `HHPS v1.3.1`

## 4. 根目录旧源码说明
- 根目录旧的 `app-*.js`、`index.js`、`host-photoshop.js` 不再是正式入口。
- 旧文件已归档到：
  - `legacy/root-src`
- 后续改动必须优先落在 `src/webview` 和 `src/host`

## 5. 当前工作台结论
- 已取消手动“放回 Photoshop”按钮。
- 当前设计是：
  - 任务完成后自动贴回 Photoshop
  - 默认允许跨文档回到源文档
  - 如果源文档存在选区，则按原选区 bounds 自动贴回并加遮罩
  - 如果源文档没有选区，则自动回贴到源文档
- 这条链路已迁移并通过构建，但真实 Photoshop 行为仍需继续验证。

## 6. 当前设置页结论
- 当前设置页不是最终定稿。
- 用户已经明确指出：
  - `2.4.0` 当前设置页布局和交互，与 `2.3.3` 不一致
- 后续仍需继续向 `2.3.3` 收敛。

## 7. 当前提示词模板结论
- 当前 `2.4.0` 已接入提示词模板第一版链路：
  - 新建
  - 编辑
  - 删除
  - 列表
  - 导入 / 导出 JSON
  - 工作台模板选择器
- 但这还不是最终体验，后续仍需继续完善：
  - 更接近 `2.3.3` 的模板交互
  - 多模板组合策略
  - 模板应用反馈
  - 模板排序 / 搜索 / 覆盖与追加行为

## 8. 当前图像输入链路结论
- 当前已完成第一版增强：
  - Photoshop 文档图像捕获
  - WebView 预览
  - 压缩参数控制
  - 多次捕获资产列表
  - 当前选中图像
  - 字段级图像绑定
  - 删除捕获时清理字段绑定
  - 一键清空全部捕获
- 后续还应继续提升：
  - 字段级上传策略可视化
  - 多图字段 / 数组字段支持
  - 更清晰的字段映射体验

## 9. 当前页面修复结论
- 本轮已经修复过一次严重页面损坏问题：
  - `app.html` 结构损坏
  - 页面串位
  - 坏闭合标签直接显示到界面
  - 文案乱码
- 当前结论：
  - 页面结构损坏问题已修复
  - 测试包已重新构建
  - 但仍要继续警惕源码里残留乱码

### 遇到乱码时必须这样处理
- 优先检查：
  - `app.html`
  - `src/webview/*.js`
  - `app.css`
- 不要只看 `release` 或 `dist`
- 不要直接手改 `dist/app.bundle.js`
- 必须修源码后重新执行：
  - `npm.cmd run build`
  - `npm.cmd run package:test`

## 10. 构建与测试
- 可用命令：
  - `npm.cmd install`
  - `npm.cmd run build`
  - `npm.cmd run package:test`
- 当前测试包目录：
  - `release/PixelRunnerV2.4.0-test`

## 11. 接手时优先顺序
1. 先阅读：
   - `docs/注意事项.md`
   - `docs/重构交接.md`
2. 继续以 `src/webview` 和 `src/host` 为正式源码开发
3. 不要回退到根目录旧源码做主开发
4. 优先把设置页与工作台交互继续向 `2.3.3` 收敛
5. 继续完善提示词模板完整链路和交互体验
6. 继续清理残留乱码与 UI 文案问题
7. 然后再继续提升图像输入链路与工程化

## 12. 一句话提醒
- 现在最重要的是“对齐 `2.3.3` 体验，同时守住 `2.4.0` 的前后端分离架构”，不要再把项目带回旧的混合式写法。
