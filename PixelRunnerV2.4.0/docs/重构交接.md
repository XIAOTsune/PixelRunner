## 3. 当前架构结论
- `2.4.0` 已经切换到 `UXP Host Shell + WebView UI` 架构。
- 必须继续坚持前后端分离和模块化。
- WebView 负责：
  - UI
  - 页面状态
  - 表单
  - 设置页交互
  - 工作台交互
  - 任务参数组织
- Host 负责：
  - bridge
  - 存储
  - RunningHub 请求
  - Photoshop / UXP 能力调用
- 不要把复杂 UI 或业务逻辑重新塞回 Host。

## 6. 当前 Host 结构
- `src/host/main.js`
  - bridge 分发入口
- `src/host/bridge.js`
  - bridge 响应与宿主事件工具
- `src/host/runninghub.js`
  - RunningHub 提交、轮询、取消、账户状态
- `src/host/photoshop.js`
  - Photoshop Host 暴露层
- `src/host/photoshop-bridge.js`
  - Host bridge 到 Photoshop service 的适配
- `src/host/photoshop/service.js`
  - Photoshop 相关核心能力
- `src/host/photoshop/document.js`
  - 文档信息、选区 bounds、文档激活等能力
- `src/host/photoshop/tool-actions.js`
  - 工具箱动作
- `src/host/photoshop/commands.js`
  - Photoshop 命令收敛

## 7. 当前 WebView 结构
- `src/webview/main.js`
  - WebView bootstrap
- `src/webview/runtime.js`
  - bridge client、storage fallback、基础工具
- `src/webview/state.js`
  - settings / apps / templates / form / task / imageCapture 等共享状态
- `src/webview/ui.js`
  - tabs、日志、通用 UI 行为、工具按钮绑定
- `src/webview/apps.js`

- `src/webview/workspace.js`

- 当前已改为：
  - 任务完成后自动贴回 Photoshop
  - 不再保留“放回 Photoshop”手动按钮
  - 不再保留手动贴回设置面板
- 当前自动贴回逻辑：
  - 记录任务发起前的源文档信息
  - 默认允许跨文档回到源文档
  - 如果源文档存在选区，则按原选区 bounds 自动贴回并加遮罩
  - 如果没有选区，则自动回贴到源文档
- 说明：
  - 代码迁移与构建已完成
  - 真实 Photoshop 行为仍需继续在插件环境中验证
