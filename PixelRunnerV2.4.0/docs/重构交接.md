# PixelRunner 2.4.0 重构交接

## 一、项目定位

- 当前项目是 Photoshop UXP 插件重构项目。
- 当前正式开发版本是 `PixelRunnerV2.4.0`。
- 功能对齐基线以 `PixelRunnerV2.3.3` 为准。
- 不再参考 `HHPS v1.3.1`，该目录已经删除，也不应再作为迁移来源。

## 三、当前架构结论

- `2.4.0` 已经从旧版单页 UXP 面板，切到 `UXP Host Shell + WebView UI` 架构。
- 必须继续坚持前后端分离和模块化，不允许把复杂 UI 或业务逻辑重新塞回宿主层。
- 当前职责边界：
  - WebView 负责 UI、状态、表单、工作台交互、设置页交互、任务参数组织。
  - Host 负责 bridge、本地存储、RunningHub 请求、Photoshop / UXP 能力调用。

  - 导入 / 导出 JSON

### 3. 工作台动态表单与任务流程

- 已完成第一版主流程：
  - 根据 `inputs` 动态渲染表单
  - 表单值采集
  - 必填校验
  - 组装 RunningHub payload
  - 提交任务
  - 轮询结果
  - 取消任务

### 4. 结果贴回 Photoshop

- 当前已改为：
  - 任务完成后自动贴回
  - 不再保留“放回 Photoshop”手动按钮
  - 不再保留手动贴回设置面板
- 当前自动贴回逻辑：
  - 记录任务发起前的源文档信息
  - 默认允许跨文档回到源文档
  - 如果源文档存在选区，则按原选区 bounds 自动贴回并加遮罩
  - 如果没有选区，则自动回贴到源文档

- 说明：
  - 这条自动贴回链路已完成代码迁移与构建验证。
  - 但由于整体迁移尚未完成，尚未把真实 Photoshop 端完整行为当作最终稳定结论。
  - 后续仍需在真实插件环境中继续验证。

### 5. 图像输入捕获与上传前处理

- 已完成第一版增强：
  - 从 Photoshop 当前文档捕获图像
  - WebView 预览

  - 多次捕获资产列表
  - 当前选中捕获图像

  - 删除单个捕获时自动清理字段绑定
  - 一键清空全部捕获
