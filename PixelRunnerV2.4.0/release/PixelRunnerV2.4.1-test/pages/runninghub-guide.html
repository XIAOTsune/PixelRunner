RunningHub 插件新手教程

这是一份专为新手准备的手把手教程。不管你是第一次接触 AI 修图插件，还是对 RunningHub 不太熟悉，都能跟着下面的 5 个步骤一步步完成配置，开始使用插件进行 AI 修图。

💡 开始之前

本教程使用的是企业共享 API Key，需要先在 RunningHub 网站注册账号，然后获取 API Key 并充值钱包余额即可使用。任务消耗的是钱包余额，非常适合新手入门。

Step 1

获取企业共享 API Key

必备步骤
1

什么是企业共享 API Key？

企业共享 API Key 是你在 RunningHub 网站注册账号后获取的 API Key。使用这个 API Key 需要先注册 RunningHub 账号。任务运行时消耗的是钱包余额，只要钱包有钱就可以持续使用。

ℹ️ 与个人 API Key 的区别

企业共享 API Key 也是你自己在 RunningHub 网站注册账号获取的，使用方式与个人 API Key 相同，主要区别在于企业共享账号可能有共享余额或其他企业级功能。

2

如何获取企业共享 API Key？

你需要在 RunningHub 官网注册账号，然后在个人设置中找到 API Key。

  • 第一步：访问 RunningHub 网站，点击注册账号
  • 第二步：完成注册并登录你的账号
  • 第三步：在个人中心或设置页面找到 API Key 选项
  • 第四步：复制你的 API Key
📋获取到的是一串类似 rh_xxxxxxxxxxxxx 这样的字符，这就是你的 API Key
3

在插件中使用 API Key

将获取到的 API Key 粘贴到插件设置中即可。API Key 很长，直接复制避免手动输入出错。

⚠️ 注意事项
  • API Key 注意不要泄露给陌生人
  • 复制时注意不要多复制或少复制字符
  • 插件会记住保存过的 API Key，不需要额外保存
  • 如果 API Key 失效或过期，需要重新到 RunningHub 网站获取
下一步：充值钱包 →

Step 2

充值钱包余额

确保有钱
1

为什么要充值钱包？

使用企业共享 API Key 运行任务时，消耗的是钱包余额。每次 AI 修图任务会根据使用的应用和图片尺寸扣除一定的费用。如果钱包余额不足，任务会失败。

💰钱包余额就像手机话费，用多少扣多少，只需要在使用前保证有钱就行
2

如何充值钱包？

在 RunningHub 官网的充值页面，可以直接使用支付宝或微信扫码充值。

前往 RunningHub 官网充值 →
3

建议充值多少？

一般任务消耗约 0.03-0.1 元/次，日常使用不需要保持太多余额：

  • 初次测试：10 元即可，够用蛮久
  • 日常使用：不需要保持大量余额，10 元够用一段时间
  • 大批量使用：可以考虑充值基础版及以上会员，享受更多优惠
✅ 小贴士

建议先用少量金额测试插件能正常工作后，再考虑充值更多。新手最容易犯的错误是：钱包没余额就开始配置插件，结果发现任务一直运行失败。

下一步：保存 API Key →

Step 3

在插件中保存 API Key

配置插件
1

打开插件设置页面

在 Photoshop 中打开 PixelRunner 插件面板，然后点击顶部的 "设置" 标签页进入设置页面。

ℹ️ 插件面板结构

插件顶部有三个标签：RH 工作台、工具箱、设置。点击"设置"即可进入配置页面。

2

找到 API Key 输入框

在设置页面中，找到 "RunningHub API Key" 输入框。界面上有一个"显示"按钮可以切换密码明文显示。

🔐输入框默认是隐藏密码状态的，点击"显示"按钮可以看到输入的内容
3

粘贴并保存 API Key

将之前获取的企业共享 API Key 粘贴到输入框中，然后点击 "保存设置" 按钮。

  • 在输入框内 右键 → 粘贴 或使用 Ctrl+V 粘贴
  • 注意不要多复制或少复制字符
  • 点击 "保存设置" 按钮确认设置
⚠️ 重要提示

保存设置后，API Key 会存储在本地浏览器中。如果清除浏览器数据，可能需要重新输入。

4

验证 API Key 是否保存成功

保存后，你可以：

  • 查看余额：保存成功后，工作台右上角会显示钱包余额
  • 尝试解析应用：可以在"应用 ID / URL"处输入一个应用 ID，点击"解析"测试
  • 观察状态：保存成功通常会有页面提示或无报错
✅ 恭喜！

API Key 保存成功后，你已经完成了插件的基础配置。接下来可以添加 AI 应用了！

下一步：添加 AI 应用 →

Step 4

添加 AI 应用

选择工具
1

什么是 AI 应用？

AI 应用是 RunningHub 平台上开发好的 AI 修图工作流。每个应用有不同的功能，比如：

  • 全能图片 Pro / 2：通用的 AI 修图功能
  • 局部重绘：只修改图片的特定区域
  • 换背景：一键替换图片背景
  • 高清修复：提升图片清晰度
  • AI 磨皮：智能美颜，平滑肌肤纹理
  • 清除杂物：一键移除图片中的不需要的物体或水印
  • 厚涂风格：将照片转换为油画或厚涂艺术风格
  • 2D 转 3D：将平面图片转换为立体 3D 效果
  • 线稿上色：自动为黑白线稿添加颜色
  • 老照片修复：修复破损、褪色的老照片
📱新手建议先从简单的应用开始，熟悉后再尝试更多功能
2

获取 AI 应用代号

群文件中有推荐的 AI 应用列表，里面包含了各个应用的代号或应用 ID。

  • 查看群文件：通常在群文件的 "AI 应用推荐" 或类似文件夹中
  • 复制代号：找到想要的应用，复制其代号（是一串字符或数字）
📋 应用代号示例

应用代号可能看起来像这样：123456789 或 app_xxxxxxxx，具体格式取决于应用类型。

3

在插件中添加应用

回到插件的设置页面，找到 "应用 ID / URL" 输入区域：

  • 在设置页面找到 "应用 ID / URL" 输入框
  • 将复制的应用代号粘贴到输入框中（如：123456 或 RunningHub 应用链接）
  • 点击旁边的 "解析" 按钮
  • 解析成功后，应用会自动保存到已保存应用列表中
💡提示：应用 ID 可以是纯数字（如 123456），也可以是完整的 RunningHub 应用链接
⚠️ 如果解析失败

请检查：1) API Key 是否已保存；2) 应用 ID 是否正确；3) 网络是否正常

4

在工作台选择应用

应用添加成功后，切换到插件的"RH 工作台"页面：

  • 点击 "选择应用" 按钮打开应用选择器
  • 在列表中找到并选择刚才添加的应用
  • 选择后，页面会显示该应用需要的参数输入框
  • 填写完参数后，点击 "运行" 按钮开始任务
✅ 应用添加成功

现在你已经可以在工作台使用这个 AI 应用了！接下来学习如何创建提示词模板，让工作更高效。

Step 5

创建和使用提示词模板

提升效率
1

什么是提示词模板？

提示词模板是预先写好的提示词内容，可以保存并在需要时快速加载。使用模板的好处：

  • 复用：同样的提示词不需要每次重新输入
  • 批量处理：可以快速切换不同的提示词跑同一批图
  • 统一风格：确保多张图片使用相同的修图风格
⚡如果你经常用同一种修图方式，强烈建议创建模板，能省很多时间
2

创建提示词模板

在插件设置页面找到 "模板标题" 和 "模板内容" 输入区域：

  • 找到 "模板标题" 输入框，给模板起一个名字（如 "高质量二次元"、"清新风格"）
  • 在 "模板内容" 文本框中输入提示词内容
  • 点击 "保存模板" 按钮完成创建
📝提示：建议单条提示词控制在 4000 字符内，插件本地不会截断模板内容
3

使用提示词模板

在工作台填写提示词参数时，可以快速加载已保存的模板：

  • 在提示词输入框中输入时，会显示提示"输入提示词或选择模板..."
  • 点击输入框右侧的模板选择器（如果有）或在设置页面选择已保存的模板
  • 模板内容会自动填充到输入框中
  • 可以根据需要微调后点击 "运行" 提交任务
💡 模板选择器

在工作台有模板模态框（templateModal），可以从中选择已保存的模板。设置页面也有已保存模板列表可以管理。

4

模板管理技巧

管理提示词模板的一些技巧：

  • 分类命名：用有意义的名字命名模板，方便快速识别
  • 定期整理：删除不用的模板，保持列表简洁
  • 版本管理：如果提示词经常调整，可以创建多个版本
  • 一键导入导出：可以将模板导出为 JSON 文件分享给别人，或导入群文件中的 JSON 提示词组
📦 导入导出模板

在设置页面的模板列表中，可以导出当前所有模板为 JSON 文件。也可以导入别人分享的 JSON 提示词组，快速获取优质模板。群文件中有很多用户分享的现成模板可供导入使用。

🎉 恭喜完成！

到这里，你已经完成了插件的全部基础配置！现在可以：

  • 在工作台选择 AI 应用
  • 填写或上传需要修图的参数
  • 使用提示词模板快速输入
  • 点击运行开始 AI 修图任务

Troubleshooting

为什么运行失败？

常见问题

最常见的 4 个原因

1. 热门应用使用人数较多，比较拥挤

全能图片 Pro / 2 这类热门应用在高峰时段更容易排队、超时或者表现不稳定，所以偶发失败并不完全是插件问题。

2. 钱包没有钱，请先查询余额

这是最容易忽略、也最常见的原因之一。如果钱包余额不足，任务会直接失败。建议随时关注钱包余额。

3. 图片涉及违规内容，未通过云端审核

这类失败通常不是参数格式问题，而是内容在云端审核阶段被拦截了。尝试换一张图片。

4. 关闭梯子或加速器

本插件使用国内网站服务器运行，梯子、加速器或某些网络代理环境有概率导致请求被屏蔽或行为异常。

建议排查顺序

先看余额

这是最容易忽略、也最常见的原因之一。

再看应用拥挤程度

尤其是全能图片 Pro / 2，热门时段容易挤。

然后检查图片内容

如果内容可能踩审核规则，建议先换一张图再试。

最后排查网络环境

把梯子、加速器、代理先关闭，再重新运行一次。

💡 一个经验

如果你看到的是"偶发失败"而不是"稳定必现失败"，优先怀疑热门应用拥挤或网络代理环境，而不是先怀疑插件本身。
